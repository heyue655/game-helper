# 常用更新命令
全量重建（代码有改动时用这个）：

docker-compose up -d --build

# 只重建某个服务（推荐，改了哪个重建哪个）：

# 只改了后端
docker-compose up -d --build backend
# 只改了 H5
docker-compose up -d --build h5
# 只改了 admin
docker-compose up -d --build admin
# 不重建，只重启容器（改了 .env 等配置时用）：

docker-compose up -d


# 系统使用说明

## 玩家介绍
**游戏玩家** 注册登录后默认为`游戏玩家`，可下订单
**打手玩家** 管理员在后台管理系统的`玩家管理`菜单中，选择指定玩家为`打手玩家`
* 普通游戏玩家下的订单可由管理员在后台管理系统的`订单管理`中，将订单分配给`打手玩家`
* `打手玩家`需通过手机端的我的任务中进行交单，交单完成后2小时内未产生投诉将进行结算分配，默认分配订单收益的80%
* `打手玩家`可以在手机端`我的收益`中查看累计收益及结算明细
* `我的收益`即将支持提现，提现至支付宝账户

## 管理员介绍
**超级管理员** admin账户拥有所有后台管理系统的权限, 可以创建普通管理员
**普通管理员** 拥有除首页统计和系统设置外的其他所有权限

## 系统介绍
**商品上传** 需管理员登录后台管理系统新增商品，新增后点击`上架`即可在手机端查看
**订单结算** 打手交单后两小时内未产生投诉，订单将自动变为待结算状态，需管理员进行计算，默认按照2/8分成，管理员可修改
**手机端配置** 为方便手机端持续更新，后台管理系统设置`BANNER管理`、`游戏管理`、`专区管理`等，游戏和专区支持管理员创建、新增、修改，BANNER即首页三个轮播图，支持随时替换修改，应当设置为合适的游戏广告图
